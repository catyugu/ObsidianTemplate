# README

## 概述

- 这是一个 Obsidian 仓库模板。
- 进行任何操作之前，运行 `python lint.py`，**若有问题，请放下手头所有工作，优先进行结构整理，修复脚本中报告的问题。**
- 如果约束存在冲突，以更深层级的约束为准。例如，当某目录 D 要求 A，而其子目录 D/E 要求非 A，则在 D/E 目录编辑时应遵守非 A 要求。
- 当用户发出指令 **“整理”** 时，你应该：
    - 对所有 `unclassified` 中的文档进行分类和归档。
    - 补全缺失的索引内容。
    - 修复所有违反约束的地方。
- 当用户发出指令 **“反思”** 时，你应该：
    - 指出已有笔记中缺漏或错误的部分。
    - 指出笔记之间潜在的可能的联系。
    - 指出可能可以归并的笔记。
    - 向用户询问是否应用全部/部分更改。
    - 执行用户批准的更改。

## `lint.py` 脚本检查的约束

- 所有文档的标题必须和文件名相同。
- 所有链接应该有效。
- 所有目录下应该有一个和目录同名的索引文件。
- 所有 `.md` 文档应该被索引文件正确索引。
- 除非特殊规定，否则禁用 emoji。
- 目录下文件数量限制（默认 20）

## 其他约束项目

- 所有文档的标题（文件名）必须有明确意义。
- 总是使用规范的 markdown 语法而非 obsidian 等扩展的语法来编辑 `.md` 文件。
- 总是用 `*` 标记加粗或斜体，用 `-` 标记无序列表。
- 所有非文本附件或者第三方内容存放在和引用到它的文本文件同级的 `attachments` 子目录下，`attachments` 中的内容仅限同级目录下的文件链接。
- 对所有文件的引用或链接应当基于从当前文件出发的相对路径（若在同一模块，或者只需要三级及以下的跳转）或者从根目录出发的绝对路径（若相隔太远，需要多于三级跳转）。

## 索引文件的内容规范

- 所有文件夹下应该有一个**和文件夹同名**的文件作为*索引*，索引文件**必须**随着文件的增删而实时更新。它应该包含以下部分：
    - **yaml 元数据**：
        - **constraints**：即编辑当前文件夹及其子文件夹时必须遵循的约束。
    - **# 标题**：当前文件夹的名称。
        - **## 概述**：大致描述当前文件夹的内容。
        - **## 文件组织**：当前目录的文件组织。它应该包含：当前目录下的所有文件（除了自己）和子目录。
            - **特别地**：`attachments` 文件夹（用于存放附件）无需加入文件组织。
